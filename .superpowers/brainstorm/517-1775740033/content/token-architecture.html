A — Figma Variables → Shared CSS file

Define tokens in Figma Variables (dark/light modes built-in). Manually copy to a shared tokens.css committed to git. Both repos import it. Simple, no extra tooling. Manual sync step when tokens change.

B — Figma Variables → JSON → Style Dictionary

Export Figma Variables to tokens.json via a plugin. Style Dictionary transforms it to CSS variables AND Tailwind config simultaneously. Both repos always in sync. Automated — change in Figma, run one command, done.

C — Dedicated babylon-tokens package

Tokens live in their own repo as an npm package. Every platform installs @babylon/tokens. Versioned releases — update at will without breaking consumers. Most scalable, most overhead. Right choice if you add a mobile app or third platform.

My recommendation: B. Style Dictionary is the industry standard for this exact problem. One JSON file is the source of truth. Figma Variables + a free plugin handles the export. You get CSS variables for bebabylon-ui and Tailwind config for babylon-app from the same source, automatically.
